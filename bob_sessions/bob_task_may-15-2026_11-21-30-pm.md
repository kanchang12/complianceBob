### Component Breakdown

#### 1. **Repository Analyzer**
- Clones GitHub repos (public or private with token)
- Parses source files (25+ languages supported)

- Extracts file structure and metadata

#### 2. **Quick Scanner** (Pattern-Based)
- Regex patterns for common violations
- Scans for hardcoded secrets, SQL injection, XSS
- Fast: ~1000 files/second
- Low false negatives, some false positives

#### 3. **RAG System** (Knowledge Base)
- Processes 10 compliance PDFs (GDPR, HIPAA, OWASP, etc.)
- Creates embeddings using Sentence Transformers
- Stores in SQLite with vector search
- Retrieves relevant context for each code snippet

#### 4. **Deep Analyzer** (AI-Powered)
- Uses Gemma 2B via Ollama
- Analyzes flagged code sections
- Augmented with RAG context
- Provides detailed explanations and remediation

#### 5. **Rules Engine**
- Maps findings to compliance frameworks
- Assigns severity levels (Critical/High/Medium/Low)
- Deduplicates and prioritizes issues
- Generates actionable recommendations

#### 6. **Report Generator**
- Executive summary (one-page)
- Detailed findings with code snippets
- Compliance matrix (which regulations are violated)
- Export formats: JSON, HTML, Markdown, Text

### RAG System Implementation

The RAG system is crucial for boosting accuracy. Here's how it works:

```python
from sentence_transformers import SentenceTransformer
import numpy as np
import sqlite3

class RAGSystem:
    """Retrieval-Augmented Generation system"""
    
    def __init__(self, db_path: str = "database/compliance.db"):
        self.embedder = SentenceTransformer('all-MiniLM-L6-v2')
        self.db_path = db_path
        self.conn = sqlite3.connect(db_path)
    
    def process_documents(self, pdf_dir: str):
        """Process compliance PDFs and create embeddings"""
        from .pdf_processor import PDFProcessor
        
        processor = PDFProcessor()
        
        for pdf_file in Path(pdf_dir).glob("*.pdf"):
            print(f"Processing {pdf_file.name}...")
            
            # Extract text from PDF
            text = processor.extract_text(pdf_file)
            
            # Chunk into manageable pieces
            chunks = self._chunk_text(text, chunk_size=1000)
            
            # Create embeddings
            embeddings = self.embedder.encode(chunks)
            
            # Store in database
            for chunk, embedding in zip(chunks, embeddings):
                self._store_embedding(
                    document_id=pdf_file.stem,
                    chunk_text=chunk,
                    embedding=embedding
                )
        
        print(f"Processed {len(list(Path(pdf_dir).glob('*.pdf')))} documents")
    
    def search(self, query: str, top_k: int = 5) -> List[str]:
        """
        Search for relevant compliance context
        
        Args:
            query: Search query (e.g., "GDPR data encryption")
            top_k: Number of results to return
        
        Returns:
            List of relevant text chunks
        """
        # Create query embedding
        query_embedding = self.embedder.encode([query])[0]
        
        # Retrieve all embeddings from database
        cursor = self.conn.cursor()
        cursor.execute("""
            SELECT chunk_text, embedding 
            FROM embeddings
        """)
        
        results = []
        for row in cursor.fetchall():
            chunk_text = row[0]
            stored_embedding = np.frombuffer(row[1], dtype=np.float32)
            
            # Calculate cosine similarity
            similarity = self._cosine_similarity(
                query_embedding, 
                stored_embedding
            )
            
            results.append((similarity, chunk_text))
        
        # Sort by similarity and return top_k
        results.sort(reverse=True, key=lambda x: x[0])
        return [chunk for _, chunk in results[:top_k]]
    
    def _cosine_similarity(self, a: np.ndarray, 
                          b: np.ndarray) -> float:
        """Calculate cosine similarity between two vectors"""
        return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
    
    def _chunk_text(self, text: str, chunk_size: int = 1000, 
                    overlap: int = 200) -> List[str]:
        """Split text into overlapping chunks"""
        chunks = []
        start = 0
        
        while start < len(text):
            end = start + chunk_size
            chunk = text[start:end]
            chunks.append(chunk)
            start += chunk_size - overlap
        
        return chunks
```

## 🚧 Challenges & Solutions

### Challenge 1: Context Window Limitations

**Problem**: Gemma 2B has an 8K token context window. Large files (10K+ lines) don't fit.

**Solution**: Implemented smart chunking strategy:
```python
def chunk_file(content: str, max_tokens: int = 6000) -> List[str]:
    """Split file into overlapping chunks"""
    lines = content.split('\n')
    chunks = []
    current_chunk = []
    current_tokens = 0
    
    for line in lines:
        line_tokens = len(line.split())
        
        if current_tokens + line_tokens > max_tokens:
            # Save current chunk
            chunks.append('\n'.join(current_chunk))
            
            # Start new chunk with overlap (last 20 lines)
            current_chunk = current_chunk[-20:] + [line]
            current_tokens = sum(len(l.split()) for l in current_chunk)
        else:
            current_chunk.append(line)
            current_tokens += line_tokens
    
    if current_chunk:
        chunks.append('\n'.join(current_chunk))
    
    return chunks
```

**Result**: Can now analyze files of any size with maintained context.

### RAG System Implementation

The RAG system is crucial for boosting accuracy. Here's how it works:

```python
from sentence_transformers import SentenceTransformer
import numpy as np
import sqlite3

class RAGSystem:
    """Retrieval-Augmented Generation system"""
    
    def __init__(self, db_path: str = "database/compliance.db"):
        self.embedder = SentenceTransformer('all-MiniLM-L6-v2')
        self.db_path = db_path
        self.conn = sqlite3.connect(db_path)
    
    def process_documents(self, pdf_dir: str):
        """Process compliance PDFs and create embeddings"""
        from .pdf_processor import PDFProcessor
        
        processor = PDFProcessor()
        
        for pdf_file in Path(pdf_dir).glob("*.pdf"):
            print(f"Processing {pdf_file.name}...")
            
            # Extract text from PDF
            text = processor.extract_text(pdf_file)
            
            # Chunk into manageable pieces
            chunks = self._chunk_text(text, chunk_size=1000)
            
            # Create embeddings
            embeddings = self.embedder.encode(chunks)
            
            # Store in database
            for chunk, embedding in zip(chunks, embeddings):
                self._store_embedding(
                    document_id=pdf_file.stem,
                    chunk_text=chunk,
                    embedding=embedding
                )
        
        print(f"Processed {len(list(Path(pdf_dir).glob('*.pdf')))} documents")
    
    def search(self, query: str, top_k: int = 5) -> List[str]:
        """
        Search for relevant compliance context
        
        Args:
            query: Search query (e.g., "GDPR data encryption")
            top_k: Number of results to return
        
        Returns:
            List of relevant text chunks
        """
        # Create query embedding
        query_embedding = self.embedder.encode([query])[0]
        
        # Retrieve all embeddings from database
        cursor = self.conn.cursor()
        cursor.execute("""
            SELECT chunk_text, embedding 
            FROM embeddings
        """)
        
        results = []
        for row in cursor.fetchall():
            chunk_text = row[0]
            stored_embedding = np.frombuffer(row[1], dtype=np.float32)
            
            # Calculate cosine similarity
            similarity = self._cosine_similarity(
                query_embedding, 
                stored_embedding
            )
            
            results.append((similarity, chunk_text))
        
        # Sort by similarity and return top_k
        results.sort(reverse=True, key=lambda x: x[0])
        return [chunk for _, chunk in results[:top_k]]
    
    def _cosine_similarity(self, a: np.ndarray, 
                          b: np.ndarray) -> float:
        """Calculate cosine similarity between two vectors"""
        return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
    
    def _chunk_text(self, text: str, chunk_size: int = 1000, 
                    overlap: int = 200) -> List[str]:
        """Split text into overlapping chunks"""
        chunks = []
        start = 0
        
        while start < len(text):
            end = start + chunk_size
            chunk = text[start:end]
            chunks.append(chunk)
            start += chunk_size - overlap
        
        return chunks
```

## 🚧 Challenges & Solutions

### Challenge 1: Context Window Limitations

**Problem**: Gemma 2B has an 8K token context window. Large files (10K+ lines) don't fit.

**Solution**: Implemented smart chunking strategy:
```python
def chunk_file(content: str, max_tokens: int = 6000) -> List[str]:
    """Split file into overlapping chunks"""
    lines = content.split('\n')
    chunks = []
    current_chunk = []
    current_tokens = 0
    
    for line in lines:
        line_tokens = len(line.split())
        
        if current_tokens + line_tokens > max_tokens:
            chunks.append('\n'.join(current_chunk))
            current_chunk = current_chunk[-20:] + [line]
            current_tokens = sum(len(l.split()) for l in current_chunk)
        else:
            current_chunk.append(line)
            current_tokens += line_tokens
    
    return chunks
```

**Result**: Can now analyze files of any size with maintained context.

### Challenge 2: Consistency Across Runs

**Problem**: AI models can give different results for the same input.

**Solution**: 
1. Set low temperature (0.1) for deterministic output
2. Use structured prompts with JSON output format
3. Implement result caching
4. Add validation layer

**Result**: 95% consistency across runs (up from 78%).

### Challenge 3: False Positives

**Problem**: Pattern matching generates many false positives.

**Solution**: Two-stage verification:
```python
def verify_finding(finding: Dict) -> bool:
    """Verify if finding is a true positive"""
    
    # Stage 1: Pattern match (fast, high recall)
    if not quick_pattern_match(finding):

    # Stage 2: AI verification (slower, high precision)

    verification = gemma_verify(
        code=finding['code'],
        context=context,

    )
    
    return verification['is_valid']
```
