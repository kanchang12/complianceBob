Which Model Is Right for You? 🤔
Gemma 4 comes in three flavors. Pick the one that fits your use case:

Gemma 4 model family spans three distinct architectures tailored for specific hardware requirements:

Small Sizes: 2B and 4B effective parameter models built for ultra-mobile, edge, and browser deployment (e.g., Pixel).
Dense: A powerful 31B parameter dense model that bridges the gap between server-grade performance and local execution.
Mixture-of-Experts: A highly efficient 26B MoE model designed for high-throughput, advanced reasoning.

# How to Start the Compliance Guardian Application

## Quick Start Command

To start the Flask API server, run:

```bash
cd compliance-guardian
python -m src.api.app
```

Or use the convenient start script:

```bash
python scripts/start_api.py
```

## Complete Setup & Run Process

### 1. First-Time Setup (One-Time Only)

```bash
# Navigate to project directory
cd compliance-guardian

# Install dependencies
pip install -r requirements.txt

# Install and start Ollama (if not already installed)
# Windows: Download from https://ollama.ai
# Mac: brew install ollama
# Linux: curl -fsSL https://ollama.ai/install.sh | sh

# Download Gemma 2B model (this happens automatically when first used)
ollama pull gemma2:2b

# Initialize database
python -c "import sqlite3; conn=sqlite3.connect('database/compliance.db'); conn.executescript(open('database/schema.sql').read()); conn.close()"

# Initialize RAG system (process regulatory PDFs)
python scripts/init_rag.py
```

### 2. Start the Application

```bash
# Start the Flask API server
python -m src.api.app

# Or use the start script
python scripts/start_api.py
```

The server will start on `http://localhost:5000`

### 3. Access the Application

Open your browser and go to:
- **Landing Page**: `file:///path/to/compliance-guardian/frontend/landing.html`
- **Main App**: `file:///path/to/compliance-guardian/frontend/index.html`
- **API Docs**: `http://localhost:5000/api/docs`

## What Happens When You Run It

1. Flask server starts on port 5000
2. API endpoints become available
3. System checks for Ollama and Gemma 2B model
4. If model not found, it will be downloaded automatically on first use
5. Frontend can connect to API and start analyzing repositories

## Model Download Details

- **Model**: Gemma 2B (gemma2:2b)
- **Size**: ~1.5 GB
- **Download Time**: 5-10 minutes (depending on internet speed)
- **Location**: Ollama manages model storage automatically
- **When**: Downloads automatically when first needed, or manually with `ollama pull gemma2:2b`

## Verification

Check if everything is working:

```bash
# Test API health
curl http://localhost:5000/api/health

# Run tests
python scripts/test_api.py
```
